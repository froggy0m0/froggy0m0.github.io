__OS : Ubuntu 18.04__

## FSB(Format String Bug)-1

### example1

```c
// Name : example1
// Compaile : gcc -o example1 example1.c -m32 -mpreferred-stack-boundary=2

#include<stdio.h>

void main()
{
	int a = 20; //hex(20) -> 0x14
	int b = 21; //hex(21) -> 0x15
	int c = 22; //hex(22) -> 0x16
	int d = 23; //hex(23) -> 0x17
	printf("%p %p %p %p %p %p %p %p\n");
}
```

__Printf__ 함수는 Printf("%d", val); 와 같이 `포맷 스트링("%d")`과  해당 포맷에 대응되는 `인자`의 형태로 사용된다.

</figure>



하지만 __example1__ 처럼 대응 되는 인자 없이 `printf("%p %p %p %p %p %p %p %p\n");` 같이 사용되었다면 __Fig1__ 과 같이 먼저 스택에 저장되었던 값들(변수a 0x14, 변수b 0x15...)등이 출력 되어 공격자가 원하는 값을 얻어낼 수 있게 될 수 있다.(포맷 스트링에서 %p로 사용되었기 때문에 변수a 값 20은 16진수로 0x14로 출력 되었다.)

그렇다고 해서 __example1__ 와 같이 의도적으로 작성된 코드가 아니라면 보기 힘들 것이다.

### example2

```c
// Name : example2
// Compaile : gcc -o example2 example2.c -m32 -mpreferred-stack-boundary=2

#include<stdio.h>

void main()
{
	int a = 30; //hex(30) -> 0x1e
	int b = 31; //hex(31) -> 0x1f
	char buf[0x20];
	gets(buf);
	printf(buf);
}
```

__example2__ 는 사용자의 입력을 받아 출력 해주는 코드이다.  위 코드도 __example1__ 과 같이 입력 값을 준다면 printf함수 호출 이전 __스택에  push 되었던 메모리 값을 읽어 올 수 있을 것이다__.

    <figcaption>Fig 2. example2 출력결과</figcaption>
</figure>



__Fig 2__ 는 __example2__ 를 실행하여 입력 값을 `AAAA %p %p %p`를 주어 실행하였다. 

__example1__ 과 같이 printf함수 호출 이전 스택에 __push__ 되었던 변수 a, b 가 출력 되었고 다음으로  AAAA(0x41414141)가 16진수형태로 출력 되었다.

그렇다면 포맷스트링을 사용하는 함수를 이용하여 __이전 스택에 저장되어있던 주소 값 이나 특정 값을 leak__ 할 수 가 있을 것이다.

### example3

```c
// Name : example3
// Compaile : gcc -o example3 example3.c -m32 -mpreferred-stack-boundary=2

#include<stdio.h>

void main()
{
	int a[] = { 108,104,111,108,101 }; // l(108), h(104), o(111), l(108), e(101)
	printf("%1$c %2$c %3$c %4$c %5$c \n\n");

	printf("%2$c %5$c %1$c %4$c %3$c\n");
}
```

__example3__ 에서 사용된 `%[숫자]$[포맷]`와 같은 형태는 printf함수 호출 시의 esp 로부터 `[숫자]` 만큼 떨어진 거리의 주소를 나타낼 수 있게 해준다. 

__example1__ 로 예를 들면 printf("%p %p %p %p %p %p %p %p\n"); 인 코드에서 __4 offset__ 떨어진 __변수d__ 만 출력을 원한다면 printf("__%4$p__ ")로 수정하게 되면  d변수의 값 0x17만 출력 되게 된다.

__Fig 3__ 은 printf함수 호출 전 스택에 push된 a배열의 값들의 오프셋을 사용해 hello를 출력했다.

스택 구조를 이해하기 위해 __example3__ 를 gdb로 실행시켰다.

main+29 부터  main+57까지 a배열에 해당하는 값들이 역순으로 스택에 push되었다. 

__Fig 4__ 를 보면 인자들이 순서대로 push 된 것처럼 보이지만 `mov    dword ptr [ebp - 0x1c], 0x6c` 로 __역순으로 push되고 있다.__ (스택에서도 역순으로 push된게 보이지만 설명하자면) 인덱스가 낮을수록(index 0 = 'l') ebp에서 멀어지고(esp와는 가깝고) 인덱스가 높아질수록(index 4 = 'e')ebp와 가까워진 것을 볼 수 있다. 배열 마지막 인덱스 에 해당하는 101은 아직 저장되지 않았지만 0xffffd35c 에 0x65로 저장되어진다.

<figure>
    <img src="https://Froggy0m0.github.io/assets/img/Pwnable/2022-01-10-fsb_32bit_part1-5.png">
    <figcaption>Fig 5. example3 GDB2</figcaption>
</figure>
GDB를 계속 실행시켜 마지막 printf인 `main+86` 까지 진행 시켰다.

main+86의 esp에서 현재 esp(0ffffd348) 로부터 계산되어 진 결과 __%1$c__ = `l`, __%2$c__ = `h`, __%3$c__ = `o`, __%4$c__ = `l`, __%5$c__ = `e`로 포맷 되어 h e l l o 라는 출력 되어 __%[숫자]$c__ 를 통해 원하는 거리에 있는 값을 얻어올 수 있었다.
