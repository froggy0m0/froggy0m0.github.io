## 라우터

>  __라우터(Router)__ 는 데이터 패킷이 목적지까지 가는 길을 결정해주는 __경로 결정(Path Determination)__ 과 결정된 경로로 데이터 패킷을 보내주는 __스위칭(Switching)__ 등의 기능을 한다.

### 라우팅  프로토콜

>  __라우팅 프로토콜(Routing Protocol)__  은 경로 결정을 하기 위해 사용되는라우팅 테이블 이웃 라우터들과 정보 교환하는 프로토콜이다.

경로 정보를 직접 지정하는 정적 경로(Static Routing) 와 자동으로 교환하는 동적 경로(Dynamic Routing Protocol) 가 있다.

#### 정적 경로

>  __정적 경로(Static Routing)__ 는 라우팅 프로토콜에서 경로를 직접 지정한다.

관리자가 직접 경로를 지정해 동적 라우팅에 비해 트래픽이 적지만, 해당 경로에 문제가 생기면 관리자가 수정할 때 까지 기다려야된다.

주로 하나의 경로로만를 통해 연결된 외부 네트워크인 스터브(Stub)네트워크 에서 사용된다.

#### 디폴트 라우트(Default Route)

>  __디폴트 라우트(Default Route)__ 는 경로를 찾지 못하거나 특정 목적지가 없는 모든 네트워크들의 경로를 정해놓은 것이다.

여러 라우팅 프로토콜에의해 경로를 찾지 못한 네트워크들은 디폴트 라우트로 지정한 경로로 간다.

디폴트 라우트는 인터넷을 사용하는 라우터, Stub 네트워크에 있는 라우터에 주로 사용된다.

디폴트 네트워크를 사용하거나 스태틱 명령을 이용해 만들 수 있다.

### 라우팅 테이블

>  __라우팅 테이블__ (Routing Table)은라우터가 전송할 데이터를 경로 배정 하기 위해 사용하는 정보이다.

라우팅 테이블에는 현재 위치에서 목적지까지의 경로 비용, 어떤 인터페이스로 가야하는지 에 대한 정보들이 생성되고, 주기적을 업데이트 되며 <bold>이웃 라우터 들과 교환이 이루어진다.</bold>

### 자율 시스템

> __자율 시스템(AS,Atuonomous System)__ 은 관리자에 의해 관리되는 네트워크와 라우터의 집합(회사,기업의 라우터 집단)이다.

AS내부의 라우팅 프로토콜은 동일해야 한다.

라우터들은 자신의 AS의 내부 네트워크 정보만 알고 있으면 된다.

  1. <bold>내부통신</bold>

     ​	동일한 자율 시스템 안에서 통신하는 경우는 __내부 라우팅 프로토콜(IRP, Interior Routing)__ 을 사용한다.

  2. <bold>외부통신</bold>

     <span>서로 다른 자율 시스템을 연결해주는 라우터 __ASBR(Autonomous System Boundary Router)__  사이에 __외부 라우팅 프로토콜(ERP, Exterior Routing Protocol)__ 을 사용한다.(RIP,IGRP,OSPF,EIGRP 등이 있다.)</span>
