## 스패닝 트리

__스패닝 트리(Spanning Tree Protocol)__ 는 _루프(loop)_ 구조를 가지는 네트워크에서 브로드캐스트가 계속 발생하는 _looping_ 을 억제하기 기능을 한다.



1. Network당 하나의 __루트 브리지(Root Bridge)__ 를 선정한다.

1. 루트 브리지가 아닌 non Root Bridge 마다 Root Bridge로 향하는 포트로 하나의 __루트 포트(root port)__ 를 선정한다.

1. Segment별로 하나의 __데지그네이티드 포트(Designated Port)__ 를 선정한다.

1. __BPDU(Bridge Protocol Data Unit)__ 을 통해서 선정한다.

   <span style="font-weight:bold">여기 까지 설정</span>

1. 출발지에서 목적지 까지 두개 이상의 경로가 있는 네트워크에서 한 개의 경로를 제외하고 모두 __Bock 상태__ 로 링크를 차단한다.

1. __브리지__ 사용하던 경로에 문제가 생기면 막았던 나머지 경로 중 하나를 열어 준다.



### 브리지 ID

__브리지ID(Bridge ID)__  는 브리지나 스위치가 통신 할 때 서로 확인하기 위해 사용한다.

16비트의 _브리지 우선순위(Bridge PRiority)_ + 48비트의 _MACAddress_ 로 구성된다.

브리지 우선순위는 아무런 설정을 하지 않은 경우 Default로 32768(16비트로 8000)

브리지 우선순위는 낮은 값이 더 높은 우선순위를 갖는다.

<span style ="font-weight:blod">스위치는 스위치 ID가 아닌 브리지 ID로 사용한다.</span>

### Path Cost 

__Path cost__ 는 목적지까지의 비용

빠를수록 더 작은 값을 가진다.

| 대역폭(Band Width) | STP Cost(Path Cost) |
| :----------------: | :-----------------: |

|      10 Mbps       |         100         |

|      45 Mbps       |         39          |

|      155 Mbps      |         14          |
|      622 Mbps      |          6          |
|       1 Gbps       |          4          |
|      10 Gbps       |          2          |

### BPDU(Bridge Protocol Data Unit)

각 브리지,스위치 들 간에 전달되는 특별한 정보교환을 위한 프레임이다.

__Root BID, Sender BID, Path Cost, Port ID__ 등 의 정보로로 구성되어있으며 스위치들은 이 __BPDU__ 를 2초마다 뿌려준다.

각 스위치들은 주고 받은 <span style="font-weight:bold">BPDU패킷의 Bridge ID, Path Cost, Port ID를 비교하여</span> 루트 브리지, 루트포트 데지그네이티드 포트가 결정된다.

#### Hello Time

루트 브리지가 주기적으로 BPDU를 보내는 시간

#### Max Age

브리지들이 루트 브리지들로 부터 일정시간 동안(Max Age) 헬로 패킷을 받지 못하면 스패닝 트리 구조를 변경하게 된다.

#### Forwarding Delay

브리지 포트가 Listening -> Learning, Learning - > Forwarding 로 다음 상태로 넘어갈때 까지 걸리는 시간.

### STP 선정

#### 선정기준 

1. Root BID 값
2. Root Bridge 까지의 Path Cost 값
3. Sender BID 값
4. Port ID

위 4기준으로 Root Bridge, Root Port Designated Port 를 선정하게된다.(낮을수록 우선순위)

#### Root Bridge

한 네트워크(브로드 캐스트 도메인)에서의 __브리지ID(Bridge Priority + MAC Address)__ 가 가장 작은 스위치가 선정 된다.

#### Root Port

Non Root Bridge 는 Root Bridge로 가는 가장 좋은 경로를 가진 포트인 __Root Port__ 을 하나씩 갖는다. 

Root Bridge까지의 __Path Cost__ 값이 작은 Port가 선정된다.

#### Designated Port

세그먼트 상에서 __Root Path Cost__ 를 비교하여 더작은 값을 가진 하나의 포트가 선정된다.

-> 루트 브리지의 모든 포트는 __Root Path Cost__ 가 0으로 모두 Designated Port가 된다 

<span style="font-weight:bold">즉, Root Bridge의 모든 포트는 Desginated Port 가 된다.</span>

위 2가지 포트(Root Port, Designated Port)에 선정되지 못한 포트들은 ND(Non Designated Port)라고 한다.

#### 선정 과정

1. 부팅을 하면 스위치나 브리지는 BPDU의 루트 브리지 ID에 자신의 BID를 넣고 내보낸다.

1. 각 브리지,스위치들은 받은 BPDU를 통해 자신이 가지고 있는 스패닝 트리의 정보와 비교해 루트 브리지를 결정한다.

1. 만약 네트워크 상에서 브리지나 스위치가 자기 자신 혼자라면 자신이 루트 브리지가 된다.

### STP 포트 상태

STP에서의 스위치나 브리지의 포트들의 상태(Disabled, Blocking, Learning, Fowarding)을 의미한다.

포트는 항상 5가지 상태 중 하나에 속하게 된다.

|    상태    |                             설명                             | 데이터 전송 | MAC Address 학습 | BPDU교환 |
| :--------: | :----------------------------------------------------------: | :---------: | :--------------: | :------: |
|  Disabled  |            포트가 고장상태거나 Shut down 인 상태             |      X      |        X         |    X     |
|  Blocking  |     부팅했거나 관리자가 Disabled 상태에서 다시 살렸을 때     |      X      |        X         |    O     |
| Listening  | Blocking 상태에서 Root Port 나 Designated Port로 선정 되었을때 |      X      |        X         |    O     |
|  Learning  | Listening 상태가 포워딩 딜레이(Fowarding Delay)동안 유지 되었을 떄 |      X      |        O         |    O     |
| Forwarding | Learning 상태가 포워딩 딜레이 동안 유지 되었을 떄(ND 포트로 변하지 않았을 떄) |      O      |        O         |    O     |

Blocking 상태에서 Root Port와 Designated Port가 되면 Listening 상태 -> Forwarding 상태 로 변한다(ND Port로 변하지 않았을 경우)

<span style="font-weight:bold">ND Port는 Blocking이 된다.</span>

### 사용하던 경로에 문제가 생기면?

1. Root Bridge는 Non Root Bridge에게 Heelo Time 마다 Hello BPDU를 보낸다.

1. Non Root Bridge는 Hello BPDU의 수신여부에 따라 경로의 상태를 확인하게 된다.

1. MAX Age 이후에도 Hello BPDU가 수신되지 않으면 Blocking 상태인 Non Designated 를 루트 포트로 세팅 한다.

1. Root Port는 Listening -> Learning -> Forwarding 과정을 거치게되고 기존 Root Port로 Forwarding상태인 포트는  Blocking 된다.
