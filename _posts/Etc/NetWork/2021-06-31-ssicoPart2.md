## 네트워킹 방식

* __LAN(근거리 통신)__

_Local Area Network_ 의 약자, 어느 한정된 공간(동일 건물, 학교, 공장 등...)에서 네트워크

* __WAN__

_Wide Area Network_ 의 약자, 멀리 떨어진 지역(광범위한)을 서로 연결하는 경우

* __이더넷(Ethernet)__

CSMA/CD 프로토콜을 사용하는 네트워킹 방식 중 하나,

* __CSMA/CD__

CSMA/CD(<span style="color:royalblue">Carrier Sense</span> <span style="color:RED">Mutiple Access</span>/ <span style="color:olive">Collision Detection</span>)

　　　　　　<span style="color:royalblue">캐리어 감지</span>　　　<span style="color:RED">다중 접근</span>　　　　<span style="color:olive">충돌 감지</span>

1. PC나 서버는 먼저 네트워크 상에 통신이 일어나고 있는 지를 확인한다.<span style="color:BLUE">Carrier Sense</span>
2. 캐리어(네트워크상의 신호)가 있다면 기다렸다가 사용한다.
3. 2개이상의 PC나 서버가 동시에 네트워크 자원을 사용하는 경우 <span style="color:RED">Mutiple Access</span>
4. 네트워크 자원을 동시 사용하여 충돌을 감지 <span style="color:Olive">Collision Detection</span>
5. Collision이 발생할 경우 랜덤한 시간 이후 재전송

* __토큰링 방식(TokenRing)__

이더넷과 같이 네트워킹 방식 중 하나로 네트워크 내부의 토큰을 사용하여 토큰을 가진 네트워크만 데이터를 전송하는 방식

## 케이블 종류

* __UTP(Unshielded Twisted Pair)__

페어가 서로 꼬여있는 케이블의 한종류(비차폐)

* __STP(Shielded Twisted Pair)__

케이블의 주위를 절연체로 감싸서 외부의 간섭을 막아주는 케이블

* __FTP__

Pair를 은박 쉴드로 감싸고 있는 케이블

* __S-FTP__

## CAT 종류

__Category1__

주로 전화망에 사용되는 용도의 케이블

최대 4Mbps의 전송능력을 가진 케이블

10 Base T 네트워크에 사용되는 케이블, 최대 10Mbps

4개의 Fair중 2개 Fair만 송수신을 위해 사용 (반 이중방식)

__Category5E__

현재 가장 많이 사용되는 규격 4개의 Pair를 사용하여 데이터 전송에 사용

__Category6__

__Category7__

기가비트 인터넷을 지원 최대 10Gbps 

## MAC Address

_Media Access Control Adrress_ 는 랜카드에 존재하여 데이터 링크 계층에서 통신을 위해 할당된 컴퓨터의 48bit의 고정된 고유주소이다.

MAC Address 는 총 12개의 16진수로 구성되어있고. 벤더, 생산자를 나타내는 앞부분은 OUI(Organizational Unique Identifier)라고 부른다. 그뒹 6자리는 호스트 식별자(Host Identifier)라고 부른다.

### ARP(Address Resolution Protocol)

네트워크상에서 IP주소를 MAC Address 로 변환시키는 프로토콜이다. 통신을 할때 상대방의 IP주소로 목적지를 정하지만 실제 데이터 이동을 위해  MAC Address 를 함께 사용한다. 때문에 APR 로 상대방 IP주소와  MAC Address 를 대응시킨다.

* 단독 네트워크, A(송신자) 와 B(수신자)가 하나의 네트워크 안에서의 통신하는경우

  1. A는 B의 IP주소를 알고 있고 브로드 캐스팅으로 B의 MAC Address를 요구한다.

  1. A가 보낸 브로드캐스트를 받은 B는 자신의 MAC Address 를 보낸다.

  1. B의 MAC Address를 알게된 이후 통신이 시작된다.

​    

* A(송신자) 와 B(수신자)가 다른 네트워크에 속한 경우

  1. A는 B의 IP주소로 B의 MAC Address 를 요구(브로드 캐스트)한다.

  1. A의 브로드캐스트에 응답이없이 라우터를 자신의 MAC Address 로 A의 메시지를 B가 속해있는 해당 네트워크로 넘겨주고

  1. B가 속해있는 네트워크의 라우터는 B의 MAC Address를 찾아보내준다.

## 캐스팅 모드 전송 방식

__유니캐스트(Unicast)__

특정 목적지의 주소 하나만을 가지고 통신 하는 방식 (일대일)

__브로드캐스트(Broadcast)__

현재 위치하고 있는 네트워크 안의 모든 네트워크 장비들에 통신하는 방식(일대다)   

​	_프레임을 받으면 PC가 CPU에 인터럽트를 걸기때문에 PC성능이 떨어진다._

ex) ARP, 라우터 끼리의 정보 교환, 다른 라우터를 찾을때

* 브로드캐스트 도메인 : 라우터에 의해서 구분된 공간

__멀티캐스트(Multicast)__

미리정해둔 그룹 멤버들에게만 보내는 방식(일대다)

__애니캐스트(Anycast)__

가장 가까운 Node와 통신하는 방식

장비 : 통신 케이블, 리피터, 허브
