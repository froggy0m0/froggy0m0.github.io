## IP

### IP주소

인터넷(TCP/IP 프로토콜)을 사용하는 각 호스트들(장비들)을 구분 하기 위해 4byte로 할당

IPv4는 이진수 32자리로  2<sup>32</sup>개 정도로(사용이 제한되는 일부를 제외) 보통 사람이 이해하기 쉽게 10진수로 8자리씩 끊어서   4자리씩 (ex 255.255.255.255) 이진수 8자리는 1옥텟, 즉4옥텟을 사용하여 표현한다 .

### 네트워크 부분과 호스트 부분

　　　　　　　　　　　　　　　네트워크가 c클래스일때

　　　　　　　　　　　　　　　_<span style="color:Olive">192</span>.<span style="color:Olive">168</span>.<span style="color:Olive">221</span>.<span style="color:BLUE">254</span>_

　　　　　　　　　　　　　　　_<span style="color:Olive; font-size:15px">네트워크부분</span>.<span style="color:BLUE; font-size:15px">호스트부분</span>_

모든 IP주소는 <span style="color:Olive">네트워크 부분(NetWork Part)</span>와 <span style="color:BLUE">호스트 부분(Host Part)</span> (또는 노드 부분이라고 하기도 함)로 나뉘어져 있다.

한 네트워크(라우터없이 통신이 가능한 브로드캐스트영역의 한 부분)에서는 <span style="color:Olive">앞에 3옥텟(네트워크 부분)</span>은 반드시 동일 해야 하고

<span style="color:Blue">나머지 1옥텟(호스트 부분)</span>은 서로 달라야 한다(1~255).

### IP주소 Class

IP주소 Class는 A,B,C,D,E 5개로 나뉜다.

구분된 클래스에 따라서 네트워크 부분과 호스트 부분을 구분하게된다.

각 클래스의 호스트 개수는 2<sup>x</sup> - 2개(네트워크 주소와 브로드캐스트 주소) 이다.

<span style="color:olive">7비트의 네트워크 부분</span>과 <span style="color:blue">24비트의 호스트 부분</span>

#### Class B

네트워크 범위 _<span style="color:Olive">128</span>.<span style="color:Olive">0</span>.<span style="color:blue">0</span>.<span style="color:BLUE">0</span>_ ~ _<span style="color:Olive">191</span>.<span style="color:Olive">255</span>.<span style="color:blue">255</span>.<span style="color:BLUE">255</span>_

#### Class C

네트워크 범위 _<span style="color:Olive">128</span>.<span style="color:blue">0</span>.<span style="color:blue">0</span>.<span style="color:BLUE">0</span>_ ~ _<span style="color:Olive">191</span>.<span style="color:blue">255</span>.<span style="color:blue">255</span>.<span style="color:BLUE">255</span>_

### 기본 게이트웨이(Default Gateway)

기본 게이트웨이(Default Gateway)란 통신을 할 때 목적지가 네트워크 내부에 존재하지 않을떄, 외부로 나가 찾게 되는데 이것이 기본 게이트웨이 즉, 라우터의 이더넷 인터페이스가 된다.

## 서브넷(Subnet) 

### 서브넷(Subnet) 

하나의 네트워크가 분할되어 나누어진 네트워크를 의미한다.

서브넷은 클래스로 분류되는 네트워크들을 적절한 크기의 네트워크로 나누어 효율적으로 관리하기 위해 생겼다.

나누어진 서브넷들은 _라우터_ 를 통해 통신한다.

### 서브넷팅(Subnetting)

네트워크를 분할하여 하나의 서브넷으로 만드는 것을 __서브넷팅(Subnetting)__ 이라고 한다.

### 서브넷마스크(Subnet Mask)

__서브넷 마스크__ 는 브로드캐스트 영역 분리, IP주소 절약, 네트워크 환경에 따라 IP주소를 나누어 서브넷을 만들기 위해 사용한다.

서브넷 마스크와 IP주소는 이진수로 변환하여 _AND_ 연산을 하는데 그렇게 연산 되어 나온 결과의 1의 부분이 네트워크 부분이다.(ex 연산된 결과가 192.168.0.0 이면 192.168이 네트워크부분)

만약 주어진 클래스를 나누어 쓰지 않아도 _디폴트 서브넷 마스크_(클래스에 따라 다르다)가 존재하고 디폴트 서브넷을 사용하지 않고 변형해서 쓴 경우를 _서브넷 마스크_ 라고 한다.

#### 서브넷 마스크 특징

1. 나누어진 (브로드캐스트 영역이 분리된) 서브넷들은 라우터를 통해 통신한다.
2. 서브넷 마스크는 이진수로 나타낼 경우 연속되는 1사이에 0이 있으면 안된다.
3. 서브넷 마스크가 사용 가능한 수는 2<sup>x </sup>- 2 (x는 호스트 부분의 비트)
