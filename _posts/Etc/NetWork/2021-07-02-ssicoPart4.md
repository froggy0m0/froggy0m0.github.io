## 랜카드(Network Interface Card)

유저가 송수신한 데이터를 케이블을 통해 허브,스위치,라우터 등의 장비로 보내주고 수신받는 데이터는 CPU로 전송하여 처리하게 한다. 

랜카드는 네트워킹방식(Ethernet, TokenRing, ATM 등...)과 노드(데스크탑,랩탑 등...), 버스방식(PCI, ISA 등...)에 의해 구분될 수 있다.



1. 랜카드는 목적지가 자신인 프레임이 들어오면 CPU에게 IRQ(Interupt ReQuest)를 이용해 인터럽트를 요청

1. CPU는 IRQ 번호를 보고 랜카드가 요청한 것을 확인

1. 미리 정해 놓은 Base Memory에서 요청된 데이터로 작업한다.

## 네트워크 장비

### 리피터(Repeater)

데이터를 단순히 증폭해주는 역할을 한다.

### 허브(Hub)

Physical layer 에서 작동하는 허브를 통해 들어오는 신호를 증폭하여 같은 허브에 연결된 각각의 PC들은 서로 통신이 가능하게해준다.(외부 인터넷X)

* __Shared 허브__ 

  전송 속도를 공유한다. 전송속도 / 노드 수

* __인테리전트 허브 (Intelligent Hub)__ 

  네트워크 관리 시스템(NMS)에서 모든 데이터를 분석, 제어, Auto Partition, 문제가 있는 포트를 Isolation 하고 조치가 가능하다.

* __더미 허브(Dummy Hub)__ 

  네트워크 관리 시스템에 의해 관리 X

* __세미 허브(Semi-intelligent Hub)__ 

  기본적으로 더미 허브지만 인텔리전트 허브에 연결되면 인텔리전트 허브가된다.

### 스위치

허브와 달리 전송 속도를 공유하지 않고 연결된 노드의 동시 통신이 가능해서 독자적인 속도를 갖는다. 따라서  허브를 사용해 클리전 도메인이 커지게 되면서 전송 속도가 저하되는 문제를 해결할수 있다.

#### Cisco스위치 MAC Address저장방식

* Dynamic 방식

  스위치가 자동으로 MAC Address 를 학습하고 학습된 주소가 디폴트 300초를 동안 사용이 없으면 삭제한다.

* Permanent 방식(Static)

  사용자가 직접 MAC Address를 입력하는 형식으로 저장되면 영구적으로 지속된다.

### 브리지

브리지는 콜리전 도메인 사이를 나누어 관리하고 MAC Address 테이블(브리지 테이블)을 통해 중간 다리 역할을 한다.

#### 브리지 & 스위치 기능

 콜리전 도메인을 나누는 능력

1. Learning

   통신하는 PC의 프레임을 받아 출발지의 MAC Address를 브리지 테이블에 학습한다.

2. Flooding

   받은 프레임의 목적지 주소가 브리지 테이블에 없는 경우 자신의 __모든 포트__ 로 뿌린다.(브로드캐스트, 멀티캐스트 경우에도)

3. Fowording

   받은 프레임의 목적지 MAC Address가 브리지 테이블에 학습되어있고, 출발지와 목적지의 세그먼트가 다른경우 __목적지에 해당하는 포트__ 에만 프레임을 뿌린다.

4. Filtering

   받은 프레임의 목적지 MAC Address와 출발지 MAC Address가 같은 세그먼트에 있는 경우 브리지를 넘어가지 않게 한다. 

   이런 기능으로 콜리전 도메인을 나누어 관리 할수있게된다.

5. Aging

   MAC Address를 브리지 테이블에서 학습한 후 Aging 타이머(Default 300s)를 작동시켜 설정된 시간이 지나면 브리지 테이블에서 삭제한다. 

   Aging 타이머가 끝나기 전에 들어온 프레임의 출발지 MAC Address와 브리지 테이블에 학습된 내용이 같다면 Aging 타이머를 리프래시(Refresh)한다. 

#### 브리지와 스위치 차이점

* 처리속도 

  __스위치__ 처리방식은 하드웨어  <span style="color:RED">></span> __브리지__ 처리방식은 소프트웨어

* 스위치는 서로 다른 전송속도를 가진 포트를 한장비로 연결이 가능하다.

* 포트수

  스위치 <span style="color:RED">></span> 브리지

* 프레임 처리 방식

  스위치 : 컷스루(Cut-through) 또는 스토어 -앤- 포워드(Store -and- forward)

  브리지 : 스토어 -앤- 포워드

  ##### 프레임 처리 방식

  1. __스토어 -앤- 포워드(Store -and- forward)__

  프레임을 모두 받아들이고, 에러가 발생되면 재전송을 요구하여 처리하는 방식

  1. __컷스루(Cut-through)__ 

  프레임의 목적지 주소(처음 48Bit)만 확인후 전송을 처리

  <span style="color:RED">장점</span> 빨리 처리하는, <span style="color:Blue">단점 </span>에러 복구능력

  1. __프래그먼트 - 프리(Fragment-free)__ 

  스토어 -앤- 포워드 방식과 컷스루 방식의 장점을 결합

  전체 프레임을 다 기다리지 않고 처음 512Bit만 확인후 전송을 처리

  컷스루 방식에 비해 에러감지 능력이 뛰어남

### Lopping

스위치나 브리지에 목적지까지의 경로가 두개 이상 존재하여 네트워크에 이상이 발생하여 프레임이 전달되지 못하고 네트워크 상에서 뻉뺑도는 현상

스패닝 트리 알고리즘(Spanning Tree Algorithm)을 통해 해결

### 스패닝 트리 알고리즘(Spanning Tree Algorithm)

Looping이 발생할수있는 두개이상의 경로가 발생하면 하나만 사용하고 나머지 하나는 기존 경로에 문제가 생길때 사용하게 한다.

### 라우터(Louter)

브로드 캐스트 도메인의 분리, 패킷 필터링, 로드 분배(경로 배정), QoS(Quality of Service) 기능 제공
