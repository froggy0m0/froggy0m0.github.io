## 가상의 랜(Virtual LAN)

<p style="font-weight:bold">하나의 스위치로 여러 네트워크를 사용하기 위해서 사용된다.</p>

하나의 포트로 여러 개의 네트워크 정보를 받고 논리적으로 브로드캐스트 도메인을 분리해 준다.

__분리된 네트워크들 간의 통신은 같은 스위치에 붙어 있더라도 라우터를 통해서 통신할 수 있다.__ 

### 트렁킹(Trunking)

<p style="font-weight:bold">하나의 포트로 다수의  VLAN정보를 보내는 방식으로 , 서로 다른 스위치에서 동일 VLAN이 있는 경우 사용된다.</p>

#### 트렁크 포트(Trunk Port)

<p style="font-weight:bold">트렁킹 설정된 여러 VLAN 에서 나오는 프레임들이 지나다니는 포트</p>

하나의 트렁크 포트에서 나오는 여러 VLAN의 프레임들은 식별을 위해 패킷에 VLAN 정보를 싣고, 이패킷들은 서로 다른 VLAN(다른 네트워크 영역)이기 때문에 라우터를 통해 통신한다.

#### 트렁킹 프로토콜(Trunking Protocol)

* ##### ISL방식(Inter-Switch Link)

  시스코에서 만든 트렁킹 프로토콜, 시스코 장비 끼리만 사용한다.

* ##### IEEE 802.1Q

  표준 트렁킹 프로토콜이고 네이티브(Natvie) VLAN을 설정할 수 있다.

  * ###### 네이티브(Native) VLAN

    ​	모든 스위치 네트워크에서 하나의 VLAN만 패킷에 정보를 붙이지 않는 방식

#### VTP(VLAN Trunking Protocol)

<p style="font-weight:bold">연결된 스위치들 끼리 VLAN 정보를 주고 받아 자동으로 동기화 하게 해주어 스위치들이 가지고 있는 VLAN 정보를 항상 일치 시켜 주는 프로토콜이다.</p>

VTP 기능은 VTP 서버에 하나만 VLAN 정보를 설정하면 트렁크 링크와 연결되어 있는 스위치 들도 자동으로 업데이트 하게 된다.

만약 VTP 기능이 없다면 VLAN 구성 변경에 따라 사용하는 스위치 모두에 VLAN 구성을 변경해야 한다.

##### 메시지 형식

1. Summary Advertisement
2. Subset Advertisement
3. Advertisement Request

##### 초기 상태에서의 디폴트

```
VTP Domain name = null
VTP Mode = Server
Config Revision = 0
VLAN = 1
```

##### VTP 모드

1. VTP 서버 모드
2. VTP 클라이언트 모드
3. VTP 트랜스페어런트 모드(Transparent Mode)
