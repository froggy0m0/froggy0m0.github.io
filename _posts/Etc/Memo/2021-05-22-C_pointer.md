## *(Asterisk)의 두가지  의미

 1. __포인터 선언 애스터리스크(Asterisk)__

    포인터 변수를 선언할 때, 변수의 메모리 주소를 저장할 때

    ex) int * ptr

 2. __역 참조(dereference) 참조 연산자__

    포인터의 메모리 주소를 역 참조하겠다 

    ex)*ptr = 30;

## 포인터 변수에는 변수의 주소 값이 들어간다.

```c
int num = 20;

int *ptr ;

ptr = &num;

printf("%p %p", &num,ptr)      //*ptr 을주지않고 ptr만 줌
    

// 010FF6F0    010FF6F0
```

즉 포인터는, 변수의 주소만 가리키며 역 참조는 주소에 접근하여 값을 가져온다.

## 왜 일반 변수가 아닌 포인터 변수를 사용해야 되는가 ?

1. 포인터 변수가 가리킬 변수가 어떤 자료형인지 명시하기 위해

1. 변수에 주소 값을 넣으면 변수가 16진수로 나타낸 숫자인지 주소 값인지 모른다

1. 포인터를 역 참조하면 선언한 자료형의 크기에 맞춰서 값을 가져오거나 저장

1. long long 타입은 8바이트 크기만큼 값을 가져오고 char타입은 1바이트만큼

```c
int num = 20;       

int ptr = &num     //잘못된 예 
//int * ptr = &num  옳바른 예
*ptr = 30;         
```

ptr이 num을 가리키는 상황에서는 num이 어떤 형태의 변수인지 ptr안에는 정보가 없음(메모리 정보가 없음)

int * ptr = &num 처럼 ptr이 가리킬 변수가 int 형이라는 명시가 있어야 한다

## 포인터 변수를 사용

1. __포인터 변수 배열 접근__

   ```c
   int * ptr = &arr[0];                    // int * ptr = arr;
   printf("%d\n", ptr[0]);
   printf("%d\n", ptr[1]);                             //인덱스접근
   ```

   

1. __포인터 연산__

   ```c
   int * ptr1 = 0x0010;
   printf("%p \n", ptr1+1);
   printf("%p \n", ptr1+2);             //ptr의 sizeof(type)만큼 주소값증가
   				
   printf("%d \n", *ptr); ptr++; 
   printf("%d \n", *ptr); ptr++;        // ptr --
   				
   //arr[i] == (*arr+i)                     arr이 배열이건 포인터건 다사용가능
   ```

## 배열과 포인터??

배열의 이름은 배열의 시작 주소 값(&arr[0] == arr[0]의 주소 값)을 의미한다.

배열은 메모리에 접속하는 * 연산이 가능하지만, 상수 형태의 포인터이기 때문에 대입 연산이 불가능하다.(주소 값의 변경이 안된다.)

```c
int arr[2] = {10,20};
int * ptr = &arr[0]; //또는 int * ptr = arr

printf("%d %d \n", ptr[0], arr[0]);	// 10  10
printf("%d %d \n", ptr[1], arr[1]); // 20  20
```

## 포인터 변수 증감,가감 연산

ptr++ 혹은 ptr1+1 연산을 통해 sizeof(type)만큼 주소 값 증감

```c
int num1 = 10;
double num2 = 20;

int* ptr1 = &num1;
double* ptr2 = &num2;

printf("ptr1 = %p ptr2 = %p \n", ptr1, ptr2);
printf("ptr1 = %p ptr2 = %p \n\n\n", ptr1+1, ptr2+1);

printf("ptr1 = %p ptr2 = %p \n", ptr1, ptr2);
ptr1++; ptr2++;
printf("ptr1 = %p ptr2 = %p \n", ptr1, ptr2);

>>
ptr1 = 010FFE54 ptr2 = 010FFE44
ptr1 = 010FFE58 ptr2 = 010FFE4C


ptr1 = 010FFE54 ptr2 = 010FFE44
ptr1 = 010FFE58 ptr2 = 010FFE4C
```

인덱스 값으로 접근 가능

```c
int arr[2] = {10,20};
int * ptr = arr;

printf("%d %d \n", *(ptr+0), *(ptr+1));
printf("%d %d \n", ptr[0], ptr[1]);

printf("%d %d \n", *(arr+0), *(arr+1));
printf("%d %d \n", ptr[0], ptr[1]);
```

## 문자열

```c
char string1[] = "string";   //문자가 저장된 배열  string1[1] == t
char * string2 = "strstr";   //문자열의 주소 값이 저장, 따라서 상수성향
```

### const 선언

```c
const int * ptr = &num;
// *ptr = 20;   컴파일 에러
```

포인터 변수를 사용하여 num의 값을 변경하는 것을 허용하지 않음

```c
int * const ptr = &num;
// ptr = &aaa;  컴파일 에러
```

ptr이 가리키는 대상의 변경을 허용하지 않음.
