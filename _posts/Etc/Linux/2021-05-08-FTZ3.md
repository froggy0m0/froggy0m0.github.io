## 콘솔과 터미널 ?

콘솔 : 서버의 로컬 장치와 직접 연결된 물리적인 1차 터미널이다, 직접 명령어를 작성할 수 있는 입출력 장치. 

터미널 : 서버(로컬 또는 원격)와 통신하여 입출력 하기위한 콘솔을 구현한 소프트웨어

두 용어모두 혼용되서 쓰이고있는듯 하지만 터미널이 더넓은 의미로 원격제어 환경까지 포함하는 의미로 사용되는듯하다.

## w

```
[trainer3@ftz trainer3]$ w
User	TTY	FROM	        LOGIN@	     IDLE	   JCPU	     PCPU	    WAHT
root	tty1	     -	             12:51pm	     0.00s	   0.20s	     0.03s      	w
guest	pts/0	210.95.24.129   03:21am	     3.10m	   0.12s	     0.01s	    -bash
 ①       ②         ③               ④            ⑤         ⑥          ⑦           ⑧
```

`w`는 서버에 어떤 사용자들이 접속해 있는지를 확인하는 명령어이다.

1. 어떤 ID로 login 했는지를 보여준다.
2. 어떤 콘솔,터미널로 접속했는지를 보여준다. tty는 콘솔 pts는 터미널을 의미한다. `/`뒤의 숫자는 접속순서에 따라 정해진다.
3. 접속한 컴퓨터의 ip를 보여준다. root의 경우 콘솔로 접속했기떄문에 ip가 나타나지 않는다.
4. 어느시간에 로그인 했는지를 보여준다.
5. 해당사용자가 마지막입력으로부터의 시간을 보여준다. 지연시간이라고한다.
6. 해당 사용자가 CPU를 사용한 지연 시간을 보여준다.
7. WHAT 필드에 있는 명령이 지연된 시간을 보여준다.
8. 해당사용자가 어떤 명령을 사용하고 있는지를 보여준다.
