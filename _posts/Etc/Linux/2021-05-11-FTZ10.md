## 리눅스의 권한

Local 해킹 : 해킹하고자 하는 서버에 일반 계정을 가지고 있을 때, __root 권한을 얻고자__ 시도하는 것

Remote 해킹 : 자신이 해킹하고자 하는 서버에 계정을 가지고 있지 않을 때, __계정을 얻고자__ 시도하는 것 

### SetUID

SetUID가 걸린 파일을 조작하여, __root 권한__ 을 얻을 수 있게 된다.

<span style="color:royalblue">Set</span><span style="color:orange">UID</span> : <span style="color:royalblue">변경하다</span> + <span style="color:orange">User ID 의 약자</span>  = __일시적으로 자신의 ID를 변경하는 것__

ex) 자신의 비밀번호를 변경 할 때, 비밀번호는 _/etc/shadow/passwd_  파일에 저장되어 root 권한만 수정할 수 있지만,

passwd파일은 __SetUID__ 가 걸려있어서 일반 사용자들이 그 passwd 파일을 실행하는 동안에는 __루트로의 일시적인 아이디 변경__ 이 된다.

​    

SetUID가 걸린 파일을 실행하면 root권한으로 변경되고,

파일의 실행이 끝나면 다시 원래의 ID로 돌아온다.

#### /usr/bin/passwd 파일 해석

```
[trainer10@ftz trainer10]$ ls -al /usr/bin/passwd
-r-s--x--x   1 root     root        22312 Sep 26  1999 /usr/bin/passwd*
[trainer10@ftz trainer10]$
```

<span style="color:royalblue">-r-</span><span style="color:orange">s</span><span style="color:royalblue">--x--x</span>   1 root     root        22312 Sep 26  1999 /usr/bin/passwd*

권한 부분에서 <span style="color:royalblue">-r-</span><span style="color:orange">s</span><span style="color:royalblue">--x--x</span>에서 x(실행 권한)의 자리에 <span style="color:orange">s</span> 가 _SetUID_ 를 의미한다. 

<span style="color:orange">s</span>는 x(실행 권한)를 포함하고 있다.

​    

 _/usr/bin/passwd_ 파일은 root에게 읽기와 실행이 있고, root라는 그룹에게는 실행 권한만 있고, 위를 제외한 사용자들에게도 실행 권한만 있다.

하지만 root에게 SetUID가 걸려있기 떄문에 어느 사용자이던간에 저 파일을 실행할 떄는 root의 권한을 갖게된다.

#### SetUID파일 찾기

```
[trainer10@ftz trainer10]$ find / -perm -4000
/bin/su
/sbin/mingetty
/usr/bin/smbpasswd
/usr/bin/sperl5.00503
...
```

___/ 에서 부터 적어도 SetUID가 걸린 모든 파일을 찾아라___ 

__\-__ : 적어도

__-perm__ : 권한을 찾겠다

__4__ : SetUID를 의미

__000__ : rwx 모두를 의미

​    

##### find 명령어옵션

* __-perm__

  권한과 일치하는 파일을 찾는다.

  _ex) find / -perm 4755_

* __-name__

  이름과 일치하는 파일을 찾는다.

  _ex) find / -name cat_

* __-user__

  유져와 일치하는 파일을 찾는다.

  _ex) find / -user mmung2_

* __-group__

  그룹과 일치하는 파일을 찾는다.

  _ex) find / -group guta_
